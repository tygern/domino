# Domino

A Go implementation of domino tableau and heap computations for type D
Coxeter groups. This is a rewrite of [BuildDomino](https://github.com/tygern/BuildDomino),
the Java application used for calculations in
[Leading Coefficients of Kazhdan-Lusztig Polynomials in Type D](https://arxiv.org/abs/1304.6074).

## Background

### Type D Coxeter groups

A [Coxeter group](https://en.wikipedia.org/wiki/Coxeter_group) $W$ is a
group generated by reflections $S = \{s_1, \ldots, s_n\}$ subject to
relations $(s_i s_j)^{m_{ij}} = e$. The **length** $\ell(w)$ of an
element $w \in W$ is the minimum number of generators in any expression
for $w$. Such a minimal expression is called a **reduced expression**. A
generator $s$ is a **right descent** of $w$ if $\ell(ws) < \ell(w)$.

The type $D_n$ Coxeter group has $n$ generators with the
[Coxeter-Dynkin diagram](https://en.wikipedia.org/wiki/Coxeter%E2%80%93Dynkin_diagram):

```
s₁
  \
   s₃ — s₄ — ··· — sₙ
  /
s₂
```

Generators $s_1$ and $s_2$ both connect to $s_3$, forming a fork.
The relations are:

- $s_i^2 = e$ for all $i$
- $s_1 s_3 = s_3 s_1$ and $s_2 s_3 = s_3 s_2$
- $s_i s_{i+1} s_i = s_{i+1} s_i s_{i+1}$ for adjacent non-branching generators
- $s_i s_j = s_j s_i$ when $|i - j| \geq 2$ and neither pair is in the branch

Type $D_n$ is realized as the subgroup of **signed permutations** — bijections
$w\colon \{\pm 1, \ldots, \pm n\} \to \{\pm 1, \ldots, \pm n\}$ with
$w(-i) = -w(i)$ — where the number of negative values in
$\{w(1), \ldots, w(n)\}$ is even. The generators act as:

- $s_1$: swap positions 1 and 2 with negation, sending $(a, b, \ldots)$ to $(-b, -a, \ldots)$
- $s_i$ ($i \geq 2$): swap positions $i{-}1$ and $i$

### Domino tableaux

A **domino tableau** is a Young-diagram-like shape filled with dominoes
($1 \times 2$ or $2 \times 1$ tiles) labeled $1$ through $n$. Labels
increase along rows and columns. The
[Garfinkle algorithm](http://www.numdam.org/item?id=CM_1990__75_2_135_0)
generalizes the
[Robinson-Schensted correspondence](https://en.wikipedia.org/wiki/Robinson%E2%80%93Schensted_correspondence)
to construct a pair of domino tableaux $(P, Q)$ from a signed
permutation $w$:

- The **right tableau** $P$ is built from $w$
- The **left tableau** $Q$ is built from $w^{-1}$
- Construction adds dominoes one at a time, each addition possibly
  triggering a **shuffle** — a sequence of twists and slides that
  rearrange existing dominoes to maintain the tableau property

The tableau correspondence gives a bijection between elements of $D_n$
and pairs of same-shape standard domino tableaux. Elements with the same
right tableau belong to the same **left cell**; elements with the same
left tableau belong to the same **right cell**. These cells are
fundamental to the representation theory of Coxeter groups.

### Heaps

A [heap](https://en.wikipedia.org/wiki/Heap_(mathematics)) is a poset
visualization of a reduced expression. Generators are represented as
blocks dropped onto the Coxeter-Dynkin diagram. Non-commuting generators
stack vertically; commuting generators sit side by side. For type $D$,
generators $s_1$ and $s_2$ share a visual column at the branch.

### Bad elements

An element $w$ is **fully commutative** if any reduced expression for
$w$ can be obtained from any other by only commutation relations
($s_i s_j = s_j s_i$). A **bad element** is one that:

1. Has no reduced expression beginning or ending in two noncommuting
   generators, and
2. Is not itself a product of commuting generators

Bad elements are precisely the problematic cases for computing the
leading coefficients $\mu(x, w)$ of
[Kazhdan-Lusztig polynomials](https://en.wikipedia.org/wiki/Kazhdan%E2%80%93Lusztig_polynomial)
in type $D$. The main result of the
[thesis](https://arxiv.org/abs/1304.6074) is that $\mu(x, w) \in \{0, 1\}$
when $x$ is fully commutative, extending a known result from type $A$.
Bad elements are the key case in the proof.

## Installation

```
go install github.com/tygern/domino/cmd/domino@latest
```

Or build from source:

```
git clone https://github.com/tygern/domino.git
cd domino
go build -o domino ./cmd/domino
```

## Usage

Elements are specified either as signed permutations (`-perm`) or as
products of generators (`-expr` with `-rank`).

### `info` — element information

Print the length, descent sets, bad status, and a reduced expression for
a signed permutation.

```shell
domino info -perm 1,-4,3,-2
```

produces

```
Permutation:   [1, -4, 3, -2]
Length:        7
Right descent: {1, 2, 4}
Left descent:  {1, 2, 4}
Bad:           true
Reduced:       (4, 1, 2, 3, 1, 2, 4)
```

You can also specify an element as a product of generators:

```shell
domino info -expr 1,2,3,4,3 -rank 4
```

produces

```
Permutation:   [-1, 4, 3, -2]
Length:        5
Right descent: {3, 4}
Left descent:  {1, 2, 4}
Bad:           false
Reduced:       (4, 1, 2, 3, 4)
```

### `tableau` — domino tableaux

Print the right and left domino tableaux as
[TikZ](https://github.com/pgf-tikz/pgf) code for LaTeX.

```shell
domino tableau -perm 1,-4,3,-2
```

produces

```
Right tableau:
\begin{tikzpicture}[node distance=0 cm,outer sep = 0pt]
\tikzstyle{ver}=[rectangle, draw, thick, minimum width=1cm, minimum height=2cm]
\tikzstyle{hor}=[rectangle, draw, thick, minimum width=2cm, minimum height=1cm]
\node[ver] at (0 + 1, 4 - 2) {2};
...
\end{tikzpicture}

Left tableau:
...
```

Include the output in a LaTeX document to render the tableaux.

### `heap` — heap diagram

Print the heap of a reduced expression as TikZ code.

```shell
domino heap -perm 1,-4,3,-2
```

produces

```
\begin{tikzpicture}[node distance=0 cm,outer sep = 0pt]
\tikzstyle{hor}=[rectangle, draw, thick, minimum width=2cm, minimum height=1cm]
\node[hor] at ( 2, 0) {4};
\node[hor] at ( 0, 0) {1\phantom{ 2}};
...
\end{tikzpicture}
```

Generators $s_1$ and $s_2$ are rendered with phantom spacing to indicate
their shared position at the branch of the Dynkin diagram.

### `bad` — enumerate bad elements

List all bad elements of $D_n$ for a given rank.

```shell
domino bad -rank 4
```

produces

```
[1, -4, 3, -2]
1 bad elements in D_4
```

```shell
domino bad -rank 6
```

produces

```
[1, -4, 3, -2, 5, 6]
[1, -4, 3, -2, 6, 5]
[-1, -6, 3, -4, 5, -2]
3 bad elements in D_6
```

The enumeration uses an optimized backtracking search with pruning and
parallel goroutines, making it feasible for larger ranks.
